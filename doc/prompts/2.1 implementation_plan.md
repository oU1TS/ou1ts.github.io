# Implementation Plan - oU1TS Root Project Website Redesign

This plan details the redesign of the "oU1TS" root project website (https://ou1ts.github.io/) into a sleek, modern Single Page Application (SPA) with responsiveness, smooth transitions, and high-fidelity aesthetics.

## Proposed Changes

### Core Architecture & UI Layout

We will update the layout of the website into a clean Single Page Application (SPA) structure:
1. **Sticky Top Navigation Bar**: Contains links to **Home**, **Projects**, and **About** sections, and a smooth **Dark/Light Theme Switcher** toggle.
2. **Home Section**:
   - Displays the community's core rationale text.
   - Prominent **"Browse Features"** CTA button that smoothly scrolls to the Projects section.
   - **Footer/Quote**: A minimalist ChatGPT quote section styled to take up exactly `100vh` at the bottom of the Home view.
3. **About Section**:
   - **Social Ecosystem Links**: Facebook, Telegram, GitHub, Reddit, and Discord styled with individual, high-fidelity platform-colored icons.
   - **Developer Attribution**: Minimalist link to creator profile: `https://github.com/b1tranger`
   - **Support Option**: Backing button/link to the support page: `https://b1tranger.bro.bd/index.html#support`
4. **Primary Projects Section**:
   - A custom, horizontal grid-like scroll section.
   - Grid constraints: Strict **3x5** viewport cells on desktop, and strict **3x3** viewport cells on mobile.
   - **Project Items**: Custom **squircle-shaped** button links (pointy corners, subtly inflated sides) created using an inline SVG `clip-path` for pixel-perfect curves.
   - **Scroll Behavior**: Infinite looping scroller with drag-to-scroll support for mouse users and standard touch scroll for mobile users. Remaining cells are filled with styled blank placeholder cells to maintain layout symmetry if there are fewer projects.
   - **Visual Effects**: Smooth linear-gradient CSS masks on left and right scroller borders to indicate overflow.

### File Changes

#### [MODIFY] [index.html](ou1ts.github.io/index.html)
- Reorganize the elements to form three clear sections: `#home`, `#projects`, and `#about`.
- Add a sticky header navigation bar `<nav>` at the top of the body containing links and a theme switcher toggle.
- Update the Home content with the specified rationale text, the "Browse Features" button, and the `100vh` footer quote.
- Implement the About section with structured developer details, support links, and colored social icons.
- Replace the old project carousel with a new horizontal grid scroll container (`.projects-scroll-wrapper` & `#projectsGrid`) and include the SVG definition for the custom squircle clip-path.

#### [MODIFY] [index.css](ou1ts.github.io/index.css)
- Implement CSS variables for Dark and Light themes to transition background gradients, cards, and text colors smoothly.
- Style the sticky navigation bar using glassmorphism.
- Set up the grid scroll container to support `grid-auto-flow: column` with 3 rows. On desktop, columns will occupy `20%` (for a 3x5 view), and on mobile, columns will occupy `33.333%` (for a 3x3 view).
- Add CSS mask-image effects on the scroller left/right edges for a faded boundary look.
- Define squircle styling using `clip-path: url(#squircle-clip)` and inner borders using `box-shadow: inset 0 0 0 1px ...` to avoid clip cutoffs.
- Add smooth transitions on theme switching and scrolling.

#### [MODIFY] [index.js](ou1ts.github.io/index.js)
- Maintain/initialize backgrounds (animated stars and floating elements). Stars will be hidden/faded in Light mode for better contrast.
- Define a list of projects dynamically (containing primary projects and other repositories).
- Implement the grid generation logic:
  - Pad the list to be a multiple of 3 and ensure a minimum of 15 items (using blank/empty cell objects).
  - Clone the columns of projects twice to create three copies (`[Copy 1][Copy 2][Copy 3]`) for infinite scrolling.
- Build the scroll loop logic:
  - Scroll the container to the start of the second (middle) copy on load.
  - Automatically jump by `copyWidth` when scroll boundaries are crossed to make scrolling infinite.
- Implement mouse drag-to-scroll mechanics.
- Bind theme toggle switch logic and sticky nav link active highlights based on intersection observers or scroll positions.

## Verification Plan

### Manual Verification
- Deploy locally and open the page in a browser.
- Verify that clicking the navigation links scrolls smoothly to the correct section.
- Verify that clicking "Browse Features" scrolls to the projects section.
- Verify that the theme switcher toggles between Dark and Light mode smoothly and styles are clean and well-contrasted.
- Verify that the horizontal projects grid displays exactly a 3x5 layout on desktop, and a 3x3 layout on mobile.
- Verify that scrolling projects in the grid loops infinitely in both directions.
- Verify that project button links are squircle-shaped with pointy corners and subtly inflated sides.
- Verify that empty slots are filled with styled blank cells.
- Verify that left and right borders of the grid scroller are faded.
- Verify that all external links open safely in a new tab (`target="_blank" rel="noopener noreferrer"`).
