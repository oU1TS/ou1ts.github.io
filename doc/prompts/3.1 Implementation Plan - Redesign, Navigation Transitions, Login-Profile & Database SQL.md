# Implementation Plan - Redesign, Navigation Transitions, Login/Profile & Database SQL

This plan outlines the changes to give the join button the same styling as the primary call-to-action button, add section transitions, implement a login/registration and user profile system in the single-page application, and define the SQL schema for Supabase integration.

## User Review Required

> [!IMPORTANT]
> - We will implement a client-side wrapper for Supabase in `index.js`. If Supabase credentials are not yet configured in a local `env-config.js`, the app will automatically fall back to a mock local database (`localStorage`) so the entire flow (Login, Register, Profile View/Edit) is fully interactive and functional in your browser out-of-the-box.
> - We will create a `doc/idea/user_profile_schema.sql` file containing the SQL schema with all requested fields.

## Open Questions

> [!NOTE]
> 1. **SQL schema constraints**: Do you want any specific validation constraints on fields like `blood_group` (e.g. enforcing valid groups like `A+`, `O-`, etc.) or `student_id` (e.g. numeric format)? Currently, the plan keeps them as flexible `TEXT` fields to prevent registration friction.
> 2. **Department choices**: For the department selection, we will provide a dropdown list containing departments like `CSE`, `EEE`, `Pharmacy`, `Civil`, `BBA`, `English`, and `Law` (common in UITS). If you'd like others, let me know.

## Proposed Changes

### Styling & Button Redesign

#### [MODIFY] [index.css](%5BoU1TS%5D/ou1ts.github.io/index.css)
- Update `.rationale-join-btn` to match the exact visual parameters of `.cta-btn` (padding, border, shadows, backdrop blur, hover translate, light mode transitions).
- Add new styles for the `#auth` and `#profile` sections:
  - Form wrapper cards styled with the same glassmorphism look as `.rationale-card`.
  - Premium-looking inputs with transition effects, focus glows, and custom labels.
  - Avatar badge rendering for user profiles.
  - Alert/notification boxes for successful login/registration or error states.

### Navigation Transitions

#### [MODIFY] [index.js](%5BoU1TS%5D/ou1ts.github.io/index.js)
- Enhance the section transition logic. In `switchTab(targetId)`, add a slight stagger or class toggle that ensures the active section triggers a clean, smooth CSS fade-in.
- Ensure that clicking navigation links correctly triggers the fade transition by removing and re-adding active state classes on the content container.

### User Login & Profile Pages

#### [MODIFY] [index.html](%5BoU1TS%5D/ou1ts.github.io/index.html)
- Add a script tag to load the Supabase client CDN (`https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2`).
- Add a script tag to load `env-config.js` for credentials.
- Add `<a href="#auth" class="nav-link" id="navAuthLink">Login</a>` inside the desktop top navigation bar (`.nav-links`).
- Add `<a href="#auth" class="sidebar-link" id="sidebarAuthLink">Login</a>` inside the mobile sidebar links (`.sidebar-links`).
- Add a new section `<section id="auth" class="spa-section">` containing:
  - Tab links to switch between Login and Register.
  - Inputs for Full Name, Email, Password, Student ID, and Department.
  - Social OAuth login option (Google OAuth).
- Add a new section `<section id="profile" class="spa-section">` containing:
  - Profile card view: Avatar, Name, Email, Student ID, Department, Batch, Blood Group, Social Links.
  - Profile edit view: Input fields to update details, dropdown for Blood Group and Department, Save/Cancel buttons.
  - A "Logout" button.

#### [NEW] [env-config.js](%5BoU1TS%5D/ou1ts.github.io/env-config.js)
- Create a configuration file containing placeholders for `SUPABASE_URL` and `SUPABASE_ANON_KEY` to support local development and prevent leaking credentials in Git.

### SQL Database Queries

#### [NEW] [user_profile_schema.sql](%5BoU1TS%5D/ou1ts.github.io/doc/idea/user_profile_schema.sql)
- Write the database schema SQL script including:
  - `CREATE TABLE public.profiles` with the new columns: `batch`, `blood_group`, `social_facebook`, `social_instagram`, `social_telegram`, `social_discord`.
  - Partial unique index for `student_id` to allow multiple OAuth users.
  - Modified trigger function `public.handle_new_user()` to populate default values during registration.
  - RLS policies to restrict updates to the profile owner and allow service-role inserts.

## Verification Plan

### Automated Tests
- Build and run check commands to ensure there are no compilation or syntax errors.

### Manual Verification
1. **Button Style**: Open `index.html` and verify that the "Join oU1TS" button in the About section visually matches the "Browse oU1TS" CTA button in size, padding, border, and hover glow.
2. **Transition**: Click various tabs in the navigation header (Home, Projects, Repos, About) and confirm that sections fade in smoothly.
3. **Login & Profile Flow**:
   - Navigate to the Login tab.
   - Register a new account. Verify registration success message.
   - Log in with the account. Verify that the navigation links change from "Login" to "Profile".
   - Navigate to the Profile tab. Verify details are displayed.
   - Click "Edit Profile", update fields (Batch, Blood Group, Facebook profile, etc.), and save. Verify details are updated.
   - Click "Logout" and confirm navigation resets to Home and links change back to "Login".
